三期项目准备 : 
	1. 检查环境配置(jdk, Tomcat, eclipse .. 版本)
	2. git仓库( git@github.com:sylvie95/Three_tzsc.git )
		1). 每个人发一个公钥给我 
			注 : 一个公钥只能一台电脑使用, 一般生成第一个默认为 id_rsa, 
				再次生成需要重新命名, 否则会覆盖第一个id_rsa, 默认的id_rsa
				建议供自己使用.
	3. 项目设计
		1). 我们开发的项目为: 跳蚤市场( 名字待定 )
			主要流程需求: 
				a. 网站主界面 : 展示一些物品, 需要显示物品的图片, 卖家报出的价格, 要求简述
					1. 商品相关图片
					2. 卖家描述信息
					3. 商品价格
					4. 商品的浏览量
					5. 发布用户的用户名
					6. 发布时间( 可显示 )
					( 需显示商品浏览量即点击量 )

				b. 用户 : 
					用户注册 --> 登录 --> 填写完善的发布详情, 发布自己想要出售的物品
					用户注册方式(手机验证)
					用户登录(以手机号码登录)
					用户联系方式(手机必填, 邮箱, QQ...)

					游客 --> 直接可点击查看商品信息及卖家要求( 注: 游客需登录之后方可查看卖家的联系方式 )

					管理员 : 进行后台管理

				c. 发布想出售的产品
					用户登录之后即可发布( 考虑是否需要管理员审核 ? )
						完善的信息包括:	
							1. 至少提供一张实体的图片信息
							2. 填写物品的描述
							3. 出售的价格

				d. 浏览需要的商品, 可查看详情
					1. 卖家(用户名)
					2. 卖家联系方式(需登录之后即可查看)
					3. 物品价格
					4. 卖家描述
					5. 项目发布时间
					6. 可进行收藏(需登录之后即可收藏)

		注意: 功能未完全, 大家想到的意见及时提出来, 进行完善
		

	4. 界面要求(一下未定, 均由参考页面选择, 参照大家意见决定)
		目前主页参考 : 慕课网主页(www.imooc.com)

任务要求: 
	1.  根据上述大体要求, 以及个人想法建议, 每人设计一份项目数据库脚本

	2.  需要大家多关注一些网页模板, 根据参照我们的上述大体需求, 
		推荐出一些页面参考模板网站( 素材准备 ), 要求每人推荐两个以上(
		不仅仅是主页, 包括一些功能界面).
	
	3. 大家发表自己的想法, 为我们的项目提供一个名字(  )
